# GitHub Deployment Guide - 0xio Wallet Recovery Tool

## Quick Deploy to GitHub Pages

### Step 1: Create GitHub Repository

```bash
cd /Users/gery/Documents/0xio/Legacy/legacy-0xio-extension

# Initialize git if not already done
git init

# Create .gitignore
cat > .gitignore << EOF
node_modules/
.DS_Store
*.log
EOF

# Add files
git add mnemonic-to-wallet.html
git add mnemonic-to-wallet.js
git add test-recovery.js
git add package.json
git add libs/nacl.min.js
git add WALLET-RECOVERY-TOOL.md
git add QUICK-START.md
git add .github-deployment.md

# Commit
git commit -m "Add 0xio wallet recovery tool"

# Create repo on GitHub first, then:
git remote add origin https://github.com/YOUR-USERNAME/0xio-wallet-recovery.git
git branch -M main
git push -u origin main
```

### Step 2: Enable GitHub Pages

1. Go to your repo on GitHub
2. Click **Settings** tab
3. Scroll to **Pages** section (left sidebar)
4. Under "Source", select: **main** branch, **/ (root)** folder
5. Click **Save**
6. Wait 1-2 minutes for deployment

### Step 3: Access Your Tool

Your tool will be available at:
```
https://YOUR-USERNAME.github.io/0xio-wallet-recovery/mnemonic-to-wallet.html
```

### GitHub Pages = Still Secure

Even though it's hosted online, the tool is STILL secure because:

1. **Client-Side Only**: All processing happens in YOUR browser
2. **No Server Communication**: Zero data sent to any server
3. **Open Source**: You can verify the code
4. **HTTPS**: GitHub Pages uses SSL/TLS

### For Maximum Security (Offline Use)

1. **Download Once:**
   ```bash
   # Download the HTML file
   ```

2. **Use Offline:**
